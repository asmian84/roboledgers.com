### Critical Fix: Bulk Dictionary Indexing
**Problem:** Bulk indexing wasn't using the Account column from CSV files to learn vendor-to-account mappings.

**Solution:** Modified `bulk-indexer.js` to correctly extract and use the `Account` column from CSV transactions.

**Impact:** When you upload historical CSV files via "Bulk Index", the system now:
- Reads the Account column (e.g., 7700, 8600, 9971)
- Cross-references transaction descriptions/payees with those account numbers
- Builds vendor dictionary entries with correct account allocations
- Learns your categorization patterns automatically

**Solution:** 
- Removed orphaned SVG elements

- Added Settings button with v1.1 preview message

**Result:** Clean, professional interface ready for production use.
